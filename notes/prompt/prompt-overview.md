# 如何编写优雅、有效的Prompt

## 1. 定义与目标
- 定义：Prompt是引导AI生成特定输出的指令
- 目标：清晰、准确、可重复地获取所需结果

## 2. 核心原则
### 2.1 清晰性
- 使用明确的语言
- 避免歧义
- 示例：
  - 差：写点关于AI的东西
  - 好：用500字解释深度学习的基本概念，面向初学者

### 2.2 结构性
- 使用层次分明的结构
- 合理分段
- 示例：
  ```
  ## 任务描述
  - 目标：创建技术路线图
  - 范围：Web前端开发
  - 输出格式：Markdown

  ## 具体要求
  1. 按技术领域分类
  2. 每个技术点包含：
     - 技术名称
     - 重要性评级（1-5星）
     - 学习资源
  ```

### 2.3 可操作性
- 提供具体参数
- 包含示例输出
- 示例：
  ```
  生成一个包含3个技术领域的路线图，每个领域列出5个技术点。
  示例输出格式：
  ## 核心知识领域
  1. 网络协议
     - HTTP/HTTPS（★★★★★）
     - WebSocket（★★★☆☆）
  ```

## 3. 最佳实践
### 3.1 角色定义
- 明确AI角色
- 示例：
  ```
  你是一个资深技术架构师，具有10年Web开发经验...
  ```

### 3.2 约束条件
- 指定输出限制
- 示例：
  ```
  输出不超过1000字，使用Markdown格式，包含3个章节...
  ```

### 3.3 迭代优化
- 添加反馈机制
- 示例：
  ```
  如果输出不符合要求，请指出具体问题并提供改进建议...
  ```

## 4. 常见问题与解决方案
### 4.1 输出不准确
- 解决方案：添加更多上下文和约束

### 4.2 输出过长
- 解决方案：明确字数限制

### 4.3 格式混乱
- 解决方案：指定输出格式

## 5. 示例模板
```markdown
## 角色定义
你是一个[角色]，具有[经验]...

## 任务描述
生成一个关于[主题]的[输出类型]...

## 具体要求
1. 包含[数量]个部分
2. 每个部分包含：
   - [要素1]
   - [要素2]
3. 输出格式：[格式]

## 示例输出
[示例内容]
```

## 6. 质量评估标准
1. 清晰度（20%）
2. 完整性（30%）
3. 可操作性（25%）
4. 可维护性（15%）
5. 可扩展性（10%）
